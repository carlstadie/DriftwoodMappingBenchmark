Statistical Improvements in This Report

1. Hierarchical Variance: All models now use hierarchical priors for sigma, improving regularization even with equal sample sizes (n=10).

2. Natural Scale Outputs: All effects are reported on both transformed and natural/interpretable scales (e.g., absolute changes, percentage changes, probability changes).

3. BF Sensitivity Analysis: Bayes factors computed with multiple KDE bandwidths (0.1, 0.3, 0.5, scott, silverman) to assess robustness.

4. ROPE Consistency: Region of Practical Equivalence (ROPE) is now applied on the transformed scale where the model lives, ensuring statistical consistency.

5. Predictive Checks: Prior and posterior predictive checks included to validate model assumptions and fit.

Interpretation Guidelines

Bayes Factors (BF10)
  • BF10 > 1 supports H1 (effect exists); BF10 < 1 supports H0 (no effect).
  • Thresholds: 1–3 anecdotal, 3–10 moderate, 10–30 strong, 30–100 very strong, >100 extreme.
  • Sensitivity: Multiple bandwidths show robustness. Consistent BF across bandwidths = stronger evidence.
  • BF is NOT a p-value; it's an evidence ratio comparing hypotheses.
ROPE (Region of Practical Equivalence)
  • ROPE is on transformed scale for consistency with model parameter space.
  • Pr(|Δ|<ROPE) = probability the effect is practically negligible.
  • High Pr(>0) with low Pr(|Δ|<ROPE) → meaningful improvement.
  • High Pr(|Δ|<ROPE) → differences are negligible.
Interpretable Effect Sizes
  • For log-scale metrics:
    • % Change: derived from exp(Δ_log) - 1
    • Absolute Change: computed via contrasts on predicted means (original scale)
  • For logit-scale metrics:
    • Odds Ratio: exp(Δ_logit)
    • Δp: difference in predicted metric (original scale) via contrasts

Understanding the factorial section
  • Effects coding: Coefficients are not simple “add to intercept” baselines.
  • Interpretable effects: We compute OR/Δp (or absolute change) from contrasts on predicted means, which is correct under effects coding.
  • BF sensitivity: If BF values are similar across bandwidths, the evidence is robust.
  • ROPE on transformed scale: For consistency with the model parameter space where effects are estimated.

  • Effects coding: Coefficients are not simple “add to intercept” baselines.

Overall multi-metric ranking

Scoring mode: rank. Rank-based utility scoring (ROBUST): For each posterior draw, converts metric values to ranks (1=best), then to utility scores where best=G, worst=1. Metric directions: Uncertainties (epistemic, aleatoric) are treated as lower-is-better (sign flipped before aggregation). Performance metrics (IoU, Dice, NSD) are higher-is-better. This ensures all metrics contribute consistently to the overall ranking.
